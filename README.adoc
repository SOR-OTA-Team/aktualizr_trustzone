= aktualizr TrustZone TA
:toc: left
:toc-title: 목차
:toclevels: 2

`sor_aktualizr`에 ARM TrustZone / OP-TEE 기반 TEE 서명 경로를 추가한 실험용 repository입니다.

Uptane OTA 클라이언트의 device private key를 Normal World 파일시스템이나 SQLite에 저장하지 않고,
Secure World TA 내부에서만 생성·보관·사용하도록 분리하는 것이 목표입니다.

[NOTE]
====
이 repository는 OP-TEE source repository가 아닙니다.
OP-TEE는 Jetson Orin Nano에서 Secure World를 구성하고 TA를 실행하기 위한 런타임으로만 사용했습니다.
====

| Jetson Linux | R36.5.0

== 아키텍처

```
┌────────────────────────────────────────┐     ┌─────────────────────────────────┐
│          Normal World (Linux)          │     │   Secure World (OP-TEE TA)      │
│                                        │     │                                 │
│  aktualizr                             │     │  aktualizr_ta                   │
│  ├── Uptane 프로토콜 처리              │     │  ├── RSA-2048 private key        │
│  ├── 메타데이터 다운로드 및 검증       │     │  │   (TEE Secure Storage)        │
│  ├── 이미지 다운로드 및 설치           │     │  ├── CMD_PROVISION_KEY           │
│  ├── SQLite storage                    │     │  │   → RSA-2048 키쌍 생성       │
│  └── keymanager.cc                     │     │  ├── CMD_GET_PUBLIC_KEY          │
│      └── TeeKeystore (kTee mode)       │     │  │   → 공개키 export (PEM)      │
│          └── tee_keystore.cc           │     │  └── CMD_SIGN_RSA_PSS            │
│              (libteec 기반 TA client)  │     │      → RSA-PSS/SHA-256 서명     │
│                                        │     │                                 │
└────────────────┬───────────────────────┘     └──────────────┬──────────────────┘
                 │          libteec / SMC                      │
                 └─────────────────────────────────────────────┘
```

Normal World는 OTA 흐름, Uptane 메타데이터 처리, 네트워크, SQLite, 이미지 다운로드·설치를 그대로 담당합니다.
Secure World는 키 관련 최소 기능만 담당합니다.

[NOTE]
====
ED25519는 구현하지 않았습니다.
aktualizr의 기존 RSA-PSS 경로를 활용하기 위해 TEE backend는 RSA-2048 전용으로 구성했습니다.
====

== TA UUID 및 Command API

=== UUID

```
a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c
```

TA 바이너리 파일명(`a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c.ta`)과
설치 경로(`/lib/optee_armtz/`) 모두 이 UUID를 사용합니다.

=== Command 목록

[cols="1,2,2,2,3"]
|===
| ID | 이름 | params[0] (INPUT) | params[1] (OUTPUT) | 설명

| `0x01` | `CMD_PROVISION_KEY` | `value.a`=key_type, `value.b`=gen_mode | — | RSA-2048 키쌍 생성 및 Secure Storage 저장 (키가 이미 있으면 no-op)
| `0x02` | `CMD_GET_PUBLIC_KEY` | `value.a`=key_type | 264-byte raw (4-byte modulus_len \| 256-byte modulus \| 4-byte exponent) | 공개키 원소 반환 (NW에서 PEM 조립)
| `0x03` | `CMD_SIGN_RSA_PSS` | 32-byte SHA-256 digest | 256-byte RSA-PSS 서명 | RSA-PSS/SHA-256 서명 (NW에서 digest 계산 후 전달)
|===

== 빌드

=== Normal World (aktualizr with TEE backend)

```bash
cmake -S . -B build-tee \
  -DBUILD_OSTREE=OFF \
  -DBUILD_P11=OFF \
  -DBUILD_TEE=ON \
  -DBUILD_SOTA_TOOLS=OFF

cmake --build build-tee --target aktualizr -j$(nproc)
```

=== Secure World (TA)

TA는 Jetson L4T R36.5 OP-TEE source package에서 생성한 `export-ta_arm64` dev kit으로 빌드합니다.

[IMPORTANT]
====
upstream OP-TEE `default_ta.pem` 기반 dev kit으로 빌드하면 TA 파일은 생성되지만
`TEEC_OpenSession()`에서 실패합니다.
반드시 *Jetson R36.5 OP-TEE source package* 기준 dev kit을 사용해야 합니다.
====

```bash
TA_DEV_KIT_DIR=<Jetson OP-TEE export-ta_arm64> \
  make -C ta CROSS_COMPILE=aarch64-linux-gnu-
```

=== TA 설치

```bash
sudo cp ta/a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c.ta /lib/optee_armtz/
```

== 테스트

=== 단위 테스트 (기존 aktualizr 테스트)

기존 keymanager/config 테스트가 TEE 수정 후에도 정상 동작함을 확인했습니다.

```bash
cmake -S . -B build-default \
  -DBUILD_OSTREE=OFF -DBUILD_P11=OFF -DBUILD_TEE=OFF -DBUILD_SOTA_TOOLS=OFF

cmake --build build-default --target t_keymanager t_config -j$(nproc)

cd build-default
ctest -R "test_keymanager|test_config" --output-on-failure
```

결과: *성공*

=== TEE Smoke test

Normal World에서 `libteec`를 통해 TA가 정상 동작하는지 검증하는 smoke test입니다.

검증 항목:

* `TEEC_InitializeContext()` / `TEEC_OpenSession()` 성공
* `CMD_PROVISION_KEY` — RSA-2048 키쌍 생성 (또는 기존 키 로드)
* `CMD_GET_PUBLIC_KEY` — 공개키 export 및 PEM 변환
* `CMD_SIGN_RSA_PSS` — RSA-PSS/SHA-256 서명 생성

실행:

```bash
sudo env LD_LIBRARY_PATH=<build>/src/libaktualizr /tmp/tee_smoke
```

성공 출력:

```
TEE smoke ok: public_key_pem=451 bytes, signature=256 bytes
```

* `public_key_pem=451 bytes`: TA에서 RSA-2048 공개키를 정상 export
* `signature=256 bytes`: RSA-2048 PSS signature가 Secure World에서 생성

== 설정

TEE key source를 사용하는 `aktualizr` 설정 예시 (`config/sota-device-cred-tee.toml` 참고):

```toml
[uptane]
key_source = "tee"
```

`key_source = "tee"` 설정 시 aktualizr는 Uptane RSA 키 생성·조회·서명을 모두 TEE backend로 위임합니다.
TEE mode에서는 private key를 SQLite `primary_keys` 테이블에 저장하지 않습니다.
