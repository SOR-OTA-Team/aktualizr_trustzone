= aktualizr_trustzone

`sor_aktualizr`에 TrustZone/OP-TEE 기반 TEE 서명 경로를 추가한 실험용 repository입니다.

이 repository는 OP-TEE source repository가 아닙니다. OP-TEE는 Jetson Orin Nano에서 Secure World를 구성하고 TA를 실행하기 위한 런타임으로만 사용했습니다.

== 목표

Uptane device private key를 Normal World 파일시스템이나 SQLite에 저장하지 않고, Secure World의 TA 내부에서만 생성/보관/사용하도록 분리했습니다.

구조는 다음과 같습니다.

----
Normal World
  aktualizr
  keymanager.cc
  tee_keystore.cc
  libteec

Secure World
  aktualizr TA
  RSA-2048 private key
  RSA-PSS/SHA-256 signing
----

Normal World는 OTA 흐름, Uptane metadata 처리, 네트워크, SQLite storage, 다운로드, 설치를 그대로 담당합니다.

Secure World는 최소 기능만 담당합니다.

* Uptane RSA-2048 private key 생성
* public key export
* RSA-PSS/SHA-256 서명

ED25519는 구현하지 않았습니다. aktualizr의 기존 RSA-PSS 경로를 최대한 활용하기 위해 TEE backend는 RSA-2048 전용으로 구성했습니다.

== 수정한 부분

=== Normal World

`CMakeLists.txt`::
`BUILD_TEE` 옵션을 추가하고, TEE 빌드 시 `libteec`를 링크하도록 수정했습니다.

`include/libaktualizr/types.h`::
`CryptoSource::kTee`를 추가했습니다.

`src/libaktualizr/utilities/config_utils.h`::
TOML 설정에서 `key_source = "tee"`를 읽을 수 있도록 수정했습니다.

`src/libaktualizr/crypto/keymanager.cc`::
Uptane RSA key 생성, public key 조회, manifest 서명을 TEE backend로 보낼 수 있도록 연결했습니다.
TEE mode에서는 private key를 SQLite `private_keys`에 저장하지 않습니다.

`src/libaktualizr/crypto/tee_keystore.h`::
Normal World에서 사용하는 TEE backend API를 정의했습니다.

`src/libaktualizr/crypto/tee_keystore.cc`::
`libteec` 기반 TA client를 구현했습니다.
TA session open, public key read, RSA-PSS signing command 호출을 담당합니다.

`src/libaktualizr/crypto/tee_keystore_dummy.cc`::
`BUILD_TEE=OFF` 빌드에서도 링크가 깨지지 않도록 dummy backend를 추가했습니다.

`src/libaktualizr/config/config_test.cc`::
`key_source = "tee"` 설정 파싱 테스트를 추가했습니다.

`config/sota-device-cred-tee.toml`::
TEE key source를 사용하는 설정 예시를 추가했습니다.

=== Secure World

`ta/aktualizr_ta.c`::
OP-TEE TA entry point와 command dispatch를 구현했습니다.

`ta/aktualizr_ta_rsa.c`::
RSA-2048 key 생성, public key export, RSA-PSS/SHA-256 signing을 구현했습니다.

`ta/aktualizr_ta.h`::
TA UUID, command ID, key ID를 정의했습니다.

`ta/user_ta_header_defines.h`::
OP-TEE TA metadata를 정의했습니다.

`ta/Makefile`, `ta/sub.mk`::
TA 빌드 구성을 추가했습니다.

TEE 옵션을 켜고 aktualizr를 빌드했습니다.

----
cmake -S . -B build-teecheck \
  -DBUILD_OSTREE=OFF \
  -DBUILD_P11=OFF \
  -DBUILD_TEE=ON \
  -DBUILD_SOTA_TOOLS=OFF

cmake --build build-teecheck --target aktualizr -j2
----

결과: 성공

== TA 빌드 및 설치

TA는 Jetson L4T R36.5 OP-TEE source package에서 생성한 `export-ta_arm64` dev kit으로 빌드했습니다.

----
TA_DEV_KIT_DIR=<Jetson OP-TEE export-ta_arm64> \
  make -C ta CROSS_COMPILE=aarch64-linux-gnu-
----

설치:

----
sudo cp ta/a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c.ta /lib/optee_armtz/
----

처음 upstream OP-TEE `default_ta.pem` 기반 dev kit으로 빌드했을 때는 TA 파일은 생성됐지만 `TEEC_OpenSession()`에서 실패했습니다.
Jetson R36.5 OP-TEE source package 기준 dev kit으로 다시 빌드한 뒤 TA open과 signing이 성공했습니다.

== TEE smoke test 구현 방식

aktualizr 전체 OTA 서버 연동 전에, TEE backend만 분리해서 검증하는 작은 smoke test를 만들었습니다.

테스트 목적:

* Normal World에서 `libteec`로 TA session을 열 수 있는지 확인
* TA가 RSA-2048 key를 생성하거나 기존 key를 읽을 수 있는지 확인
* Secure World에서 public key를 export할 수 있는지 확인
* Secure World에서 RSA-PSS/SHA-256 signature를 생성할 수 있는지 확인

테스트 프로그램의 핵심 흐름:

----
TeeKeystore::IsAvailable()
  -> TEEC_InitializeContext()
  -> TEEC_OpenSession()

TeeKeystore::ReadUptanePublicKey()
  -> TA CMD_GET_PUBLIC_KEY
  -> key가 없으면 TA CMD_PROVISION_KEY
  -> RSA public key PEM 반환

TeeKeystore::SignRsaPssSha256("{\"hello\":\"tee\"}")
  -> Normal World에서 SHA-256 digest 생성
  -> TA CMD_SIGN_RSA_PSS 호출
  -> Secure World에서 RSA-PSS 서명
  -> 256-byte signature 반환
----

실행:

----
sudo env LD_LIBRARY_PATH=$HOME/Desktop/sor_aktualizr/build-teecheck/src/libaktualizr /tmp/tee_smoke
----

성공 결과:

----
TEE smoke ok: public_key_pem=451 bytes, signature=256 bytes
----

의미:

* `public_key_pem=451 bytes`: TA에서 RSA public key를 정상 export했습니다.
* `signature=256 bytes`: RSA-2048 PSS signature가 Secure World에서 생성됐습니다.
* 따라서 aktualizr의 TEE backend가 실제 TA를 통해 key access와 signing을 수행함을 확인했습니다.
