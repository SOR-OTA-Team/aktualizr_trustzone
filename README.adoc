= aktualizr TrustZone TA
:toc: left
:toc-title: 목차
:toclevels: 2
:source-highlighter: rouge

ARM TrustZone 기반 Trusted Application (TA) 로 Uptane OTA 클라이언트의 device private key 를
Secure World 에서만 생성·보관·사용하도록 분리하는 *`sor_aktualizr` overlay* 입니다.

[cols="1,3"]
|===
| 목표        | Uptane 서명 키를 Normal World 디스크(SQLite) 와 메모리에서 제거하고 TrustZone TA 안으로 이동
| 대상 환경   | NVIDIA Jetson Orin Nano + Jetson Linux R36.5 + `nvidia-l4t-optee`
| 알고리즘    | RSA-2048 / RSA-PSS-SHA256 (aktualizr 기존 RSA-PSS 경로 재사용)
| 통합 방식   | Overlay — `ta/` + `nw/` 신규 파일 + `patches/` upstream 수정
| 검증 상태   | TA + libteec 경로 동작 확인 (smoke test). end-to-end OTA 연결은 미완료
|===

[NOTE]
====
TrustZone / OP-TEE / TA 개념이 처음이라면 먼저 link:docs/trustzone-primer.adoc[`docs/trustzone-primer.adoc`]
부터 읽으면 본 repository 의 구조와 호출 흐름을 더 쉽게 이해할 수 있습니다.
====

== 동기

aktualizr 기본 구성에서 Uptane device key 와 TLS client key 는 모두 `/var/sota/sql.db`
SQLite DB 에 평문으로 저장됩니다. NW Linux 가 root 권한으로 침해되면 *두 키 모두 디스크에서 그대로 추출* 됩니다.

이 repository 는 RSA-2048 Uptane 서명 키를 OP-TEE TA Secure Storage 로 옮겨,
침해된 NW 가 키 사본을 추출하거나 임의 데이터에 서명을 받을 수 없도록 만듭니다.

상세 위협 모델: link:docs/trustzone-primer.adoc#_ota_시나리오에서_trustzone_의_가치[primer §4]

== 아키텍처

```
Normal World (Linux)                          Secure World (OP-TEE)
─────────────────────────────────             ─────────────────────────────────
aktualizr                                     aktualizr_ta  (UUID a6f79d6e-…)
  ├── Uptane 프로토콜                           ├── RSA-2048 private key
  ├── 메타데이터 검증                           │   (TEE Secure Storage)
  ├── 이미지 다운로드 + 설치                    │
  ├── SQLite storage (키 제외)                  ├── CMD_PROVISION_KEY
  └── KeyManager (kTee mode)                    │     RSA-2048 keypair 생성
        └── nw/tee_keystore.cc                  │
              libteec client                    ├── CMD_GET_PUBLIC_KEY
                  │                             │     공개키 raw export
                  │                             │
                  │   libteec → optee 드라이버  └── CMD_SIGN_RSA_PSS
                  │   → SMC                            RSA-PSS / SHA-256 서명
                  └──────────────────►
```

설계 원칙:

* *Trusted Computing Base (TCB) 최소화* — SW 에는 키와 암호 연산만. TLS 스택·메타데이터 검증·네트워크는 NW 에 유지
* *키 비노출* — private key 는 `TEE_GenerateKey()` 로 SW 안에서 생성. NW 는 핸들도 받지 않음
* *Semantic Signing 가능 구조* — TA 가 입력 buffer 의 의미를 검증한 뒤에만 서명 (현재 RSA TA 는 SHA-256 digest 만 받으므로 NW 가 sign payload 를 통제. 추후 manifest JSON 직접 전달 방식으로 강화 가능)

== TA 명령 API

UUID: `a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c`

[cols="1,2,3,3"]
|===
| ID | 이름 | `params[0]` (input) | `params[1]` (output)

| `0x01` | `CMD_PROVISION_KEY`  | `value.a` = key_type, `value.b` = gen_mode | —
| `0x02` | `CMD_GET_PUBLIC_KEY` | `value.a` = key_type                        | 264 byte raw `len(4) ‖ modulus(256) ‖ exponent(4)`
| `0x03` | `CMD_SIGN_RSA_PSS`   | 32 byte SHA-256 digest                     | 256 byte RSA-PSS 서명
|===

* `CMD_PROVISION_KEY` 는 키가 이미 존재하면 no-op
* `CMD_GET_PUBLIC_KEY` 는 raw modulus/exponent 만 반환. PEM 변환은 NW `tee_keystore.cc` 가 처리
* `CMD_SIGN_RSA_PSS` 는 NW 가 SHA-256 digest 를 미리 계산해 전달. 알고리즘은 `TEE_ALG_RSASSA_PKCS1_PSS_MGF1_SHA256`

호출 시 `TEE_Param[4]` 구조와 NS bit 전환 메커니즘 설명은 link:docs/trustzone-primer.adoc#_trusted_application_모델[primer §3] 참조.

== 사용 방법

=== 1. OP-TEE 환경 준비

Jetson Orin Nano + Jetson Linux R36.5 기준 환경 구성은 link:docs/jetson-setup.adoc[`docs/jetson-setup.adoc`] 에 정리되어 있습니다.

핵심 요구사항:

```bash
sudo apt install nvidia-l4t-optee libteec-dev
ls /dev/tee0 /dev/teepriv0       # 디바이스 노드 확인
systemctl status tee-supplicant  # 데몬 동작 확인
```

=== 2. TA 빌드 및 설치

[IMPORTANT]
====
반드시 *Jetson R36.5 OP-TEE source package* 에서 생성한 Dev Kit 으로 빌드해야 합니다.
upstream `default_ta.pem` 으로 빌드한 TA 는 Jetson 에서 `TEEC_OpenSession()` 시 거절됩니다.
====

[source,bash]
----
cd ta
TA_DEV_KIT_DIR=<Jetson export-ta_arm64> \
  make CROSS_COMPILE=aarch64-linux-gnu-

sudo cp a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c.ta /lib/optee_armtz/
----

=== 3. aktualizr overlay 적용

`sor_aktualizr` 작업 트리에서:

[source,bash]
----
cd <sor_aktualizr>

# 신규 파일 복사
cp <aktualizr_trustzone>/nw/tee_keystore.h        src/libaktualizr/crypto/
cp <aktualizr_trustzone>/nw/tee_keystore.cc       src/libaktualizr/crypto/
cp <aktualizr_trustzone>/nw/tee_keystore_dummy.cc src/libaktualizr/crypto/

# 설정 예시
cp <aktualizr_trustzone>/config/sota-device-cred-tee.toml config/

# upstream 수정 패치 적용
git apply <aktualizr_trustzone>/patches/0001-aktualizr-tee-backend.patch
----

=== 4. aktualizr 빌드

[source,bash]
----
cmake -S . -B build-tee \
  -DBUILD_OSTREE=OFF \
  -DBUILD_P11=OFF \
  -DBUILD_TEE=ON \
  -DBUILD_SOTA_TOOLS=OFF

cmake --build build-tee --target aktualizr -j$(nproc)
----

=== 5. 디바이스 설정

`/etc/aktualizr/aktualizr.toml`:

[source,toml]
----
[uptane]
key_source = "tee"
key_type = "RSA2048"
----

`key_source = "tee"` 일 때 aktualizr 는 Uptane 키 생성·조회·서명을 모두 TA 로 위임하며,
SQLite `primary_keys` 테이블에 private key 를 *저장하지 않습니다*.

----

|===
