= aktualizr TrustZone TA
:toc: left
:toc-title: 목차
:toclevels: 2

`sor_aktualizr` 에 ARM TrustZone / OP-TEE 기반 TEE 서명 경로를 추가하는 overlay repository 입니다.

Uptane OTA 클라이언트의 device private key 를 Normal World 파일시스템이나 SQLite 에 두지 않고,
Secure World TA 내부에서만 생성·보관·사용하도록 분리합니다.

[NOTE]
====
이 repository 는 `aktualizr` fork 가 아니라 *overlay* 입니다.

`ta/` 는 그 자체로 빌드 가능한 OP-TEE Trusted Application 이고,
`nw/` 와 `patches/` 는 upstream `sor_aktualizr` 에 적용하면 TEE backend 가 활성화됩니다.
====

== 아키텍처

```
Normal World (Linux)                          Secure World (OP-TEE)
─────────────────────────────────             ─────────────────────────────────
aktualizr                                     aktualizr_ta  (UUID: a6f79d6e-…)
├── Uptane 프로토콜                           ├── RSA-2048 private key
├── 메타데이터 검증                           │   (TEE Secure Storage)
├── 이미지 다운로드 + 설치                   │
├── SQLite storage                            ├── CMD_PROVISION_KEY
└── keymanager.cc                             │   → RSA-2048 keypair 생성
    └── TeeKeystore (kTee mode)               │
        └── nw/tee_keystore.cc                ├── CMD_GET_PUBLIC_KEY
            (libteec client)                  │   → 공개키 export (raw)
                  │                           │
                  │   libteec / SMC           └── CMD_SIGN_RSA_PSS
                  └─────────────────►             → RSA-PSS / SHA-256 서명
```

Normal World 는 OTA 흐름·메타데이터·네트워크·SQLite·다운로드·설치를 그대로 담당합니다.
Secure World 는 RSA-2048 키 생성, 공개키 export, RSA-PSS 서명만 담당합니다.

[NOTE]
====
ED25519 는 구현하지 않았습니다.
aktualizr 의 기존 RSA-PSS 경로를 활용하기 위해 TEE backend 는 RSA-2048 전용입니다.
====

== TA 명령 API

UUID: `a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c`

[cols="1,2,3,3"]
|===
| ID | 이름 | params[0] (in) | params[1] (out)

| `0x01` | `CMD_PROVISION_KEY` | `value.a` = key_type, `value.b` = gen_mode | —
| `0x02` | `CMD_GET_PUBLIC_KEY` | `value.a` = key_type | 264 byte: `len(4)` ‖ `modulus(256)` ‖ `exponent(4)`
| `0x03` | `CMD_SIGN_RSA_PSS` | 32 byte SHA-256 digest | 256 byte RSA-PSS 서명
|===

* `CMD_PROVISION_KEY` 는 키가 이미 있으면 no-op
* `CMD_GET_PUBLIC_KEY` 는 raw modulus/exponent 만 반환. PEM 변환은 NW (`tee_keystore.cc`) 에서 처리
* `CMD_SIGN_RSA_PSS` 는 NW 에서 SHA-256 digest 를 계산한 뒤 전달. TA 내부 알고리즘은 `TEE_ALG_RSASSA_PKCS1_PSS_MGF1_SHA256`

== 사용 방법

=== 1. TA 빌드

OP-TEE 환경 구성 및 Dev Kit 빌드는 link:docs/jetson-setup.adoc[`docs/jetson-setup.adoc`] 참고.

```bash
cd ta
TA_DEV_KIT_DIR=<Jetson OP-TEE export-ta_arm64> \
  make CROSS_COMPILE=aarch64-linux-gnu-

sudo cp a6f79d6e-1b2c-4f8a-9bc3-d1e5f2a87b4c.ta /lib/optee_armtz/
```

[IMPORTANT]
====
반드시 *Jetson R36.5 OP-TEE source package* 기준 Dev Kit 으로 빌드해야 합니다.
upstream `default_ta.pem` 으로 빌드한 TA 는 Jetson 에서 `TEEC_OpenSession()` 실패합니다.
====

=== 2. aktualizr overlay 적용

`sor_aktualizr` 작업 디렉터리에서:

```bash
cd <sor_aktualizr>

# 신규 파일 복사
cp <aktualizr_trustzone>/nw/tee_keystore.h         src/libaktualizr/crypto/
cp <aktualizr_trustzone>/nw/tee_keystore.cc        src/libaktualizr/crypto/
cp <aktualizr_trustzone>/nw/tee_keystore_dummy.cc  src/libaktualizr/crypto/

# 설정 예시 복사
cp <aktualizr_trustzone>/config/sota-device-cred-tee.toml  config/

# 기존 파일 수정 패치 적용
git apply <aktualizr_trustzone>/patches/0001-aktualizr-tee-backend.patch
```

패치가 수정하는 파일:

* `CMakeLists.txt` — `BUILD_TEE` 옵션 + libteec 링크
* `include/libaktualizr/types.h` — `CryptoSource::kTee`
* `src/libaktualizr/utilities/config_utils.h` — `key_source = "tee"` 파싱
* `src/libaktualizr/crypto/CMakeLists.txt` — tee_keystore 빌드 등록
* `src/libaktualizr/crypto/keymanager.cc` — TEE backend dispatch
* `src/libaktualizr/config/config_test.cc` — 설정 파싱 테스트

=== 3. aktualizr 빌드

```bash
cd <sor_aktualizr>
cmake -S . -B build-tee \
  -DBUILD_OSTREE=OFF \
  -DBUILD_P11=OFF \
  -DBUILD_TEE=ON \
  -DBUILD_SOTA_TOOLS=OFF

cmake --build build-tee --target aktualizr -j$(nproc)
```

=== 4. 설정 적용

`/etc/aktualizr/aktualizr.toml` 에 다음 추가:

```toml
[uptane]
key_source = "tee"
key_type = "RSA2048"
```

`key_source = "tee"` 일 때 aktualizr 는 Uptane RSA 키 생성·조회·서명을 모두 TA 로 위임하며,
private key 를 SQLite `primary_keys` 테이블에 저장하지 않습니다.

== 테스트 결과

=== aktualizr 단위 테스트

기존 `t_keymanager` / `t_config` 가 overlay 적용 후에도 통과:

```
test_keymanager  PASS
test_config      PASS
```

=== TEE smoke test

`libteec` → TA 전 경로 검증:

```
TEE smoke ok: public_key_pem=451 bytes, signature=256 bytes
```

* `public_key_pem=451 bytes` — TA 가 RSA-2048 공개키를 정상 export
* `signature=256 bytes` — Secure World 에서 RSA-PSS 서명 생성
